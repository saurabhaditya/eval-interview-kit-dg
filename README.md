# AI Agent Eval System — Interview Exercise

Welcome! In this exercise you'll read a technical article about building evaluations for AI agents, then use your preferred AI coding tools to build an eval system.

## Time Breakdown

| Phase | Duration |
|-------|----------|
| Silent reading | ~10 min |
| Build + discussion | ~30 min |
| Wrap-up | ~5 min |
| **Total** | **~45 min** |

## What's in this repo

## Exercise Flow

1. **Read** `reading-material.md` — a condensed version of Anthropic's "Demystifying Evals for AI Agents"
2. **Build** a general-purpose eval system based on the concepts from the reading
3. **Discuss** tradeoffs, grader design, and architecture decisions with the interviewer

## How You'll Be Evaluated

We're not testing your ability to write code by hand — we want to see how you **orchestrate AI tools** to bring a technical concept to life. Specifically:

- **Technical synthesis** — Can you read a new concept and decompose it into buildable parts?
- **System design** — Do you split the system along natural boundaries (scenarios, graders, harness, reporting)?
- **AI tool orchestration** — Do you effectively direct AI coding agents, run them in parallel, and iterate on their output?
- **Quality awareness** — Do you verify your work as you go (tests, QA, reviewing agent output)?
- **Completeness** — Does the system work end-to-end by the end of the session?

Use whatever language, framework, and AI tools you're most productive with. There are no wrong choices here.

Good luck!
